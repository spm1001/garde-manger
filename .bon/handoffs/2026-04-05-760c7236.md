## For Claudes to come

When integrating a shared library into a module that previously did everything internally, the position-mapping contract is the hidden coupling. Garde's `full_text()` and `messages_with_offsets()` were tightly coupled by co-iteration — both walked the same `messages` list in the same order, so offsets were correct by construction. Delegating `full_text()` to deglacer broke that coupling: now `messages_with_offsets()` must reverse-engineer positions in someone else's output via string searching. This works but is fragile by design. The lesson: when two functions must agree on output format, compute them together (single-pass) or make the format a tested contract. Coupling-by-coincidence (searching for `── HUMAN ──` headers) is a maintenance trap. This pattern will recur anywhere garde wraps an external library's output and needs structural metadata about it.
