## For Claudes to come

**Two intertwined insights about cross-repo hygiene as diagnostic.**

**(1) The hygiene sweep IS the diagnostic for silent-success in CI.** When CI's pass/fail signal can't be trusted (the failure mode leaves "completed" status intact, the alerting has gaps, the post-step error doesn't surface), the *real* signal is end-to-end product comparison: does the consumer have what the source produces? In this session, the assemble workflow had been silently failing every Monday for 5+ weeks — exit code 1, no PR opened, no alert. The drift only surfaced when I compared mise-en-space's source version (0.7.2) to the assembly's mirror (0.5.13). The pattern from self.md ("don't trust the layer of monitoring that the failure mode would also leave intact") generalises here: for any pipeline with intermediate steps, the real diagnostic is the terminal output, not the workflow status. The 17-day cadence on these sweeps is doing real work, not hygiene theatre — it's the ground-truth check that backstops every assumed-working automation.

**(2) Verify which distribution path is in production use before reasoning about install-time bugs.** Repos can have multiple install channels (marketplace clone-from-URL vs `.mcpb` pack-and-sideload vs git clone vs published package), and a real bug in one path doesn't necessarily affect users on another. roopzRMC's `.mcpb` install repro for mise-en-space was real, complete, end-to-end tested. I extrapolated to "this affects Desktop installs" without checking that Sameer's actual install path (marketplace) clones the source repo directly via `marketplace.json` URL source and never touches `.mcpbignore` at all. The verification is cheap: read `marketplace.json`'s `source` field, check `gh release list` for published artifacts, look at `~/.claude/plugins/cache/` to see what's actually deployed. Two minutes saves you from confidently misreporting impact. The general form: when a repo has multiple distribution mechanisms, install-time bugs are scoped to specific paths, not universal — and the path you're imagining might not be the path your user is on.
