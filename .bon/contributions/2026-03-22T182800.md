garde-manger's architecture has a deliberate split between "indexing" (fast, free, happens at session end) and "extraction" (slow, LLM-powered, happens in backfill or via staged JSON from /close). The session-end hook must never call claude -p — that's the fork bomb path. The `ingest-session` command is the enforcement point: it only does indexing and staged-extraction storage, never LLM calls. If you're tempted to add extraction logic to the session-end path, resist — that's what backfill is for. The staged extraction from /close is the only way to get rich extraction without waiting for backfill, and it works because the generating Claude already has full session context — no subprocess needed.
